## Security Overview

We treat your email data with bank-level security standards.

## Encryption

**At Rest**

- AES-256 encryption for all stored data
- AWS KMS with automatic key rotation
- Encrypted backups across multiple regions

**In Transit**

- TLS 1.3 for all data transmission
- Perfect Forward Secrecy for all sessions
- 2048-bit RSA certificates

## Infrastructure

**Cloud Security**

- Hosted on AWS with multi-region redundancy
- Isolated VPCs per environment
- 24/7 automated threat detection
- DDoS protection with AWS Shield

**Database Protection**

- Encrypted at rest
- IP whitelisted access only
- Not accessible from public internet
- Automated encrypted backups

## Authentication

**OAuth 2.0**

- Sign in via Google/Microsoft (no password to steal)
- Tokens expire regularly
- Securely encrypted token storage

**Two-Factor Authentication**

- Available for all accounts
- TOTP-based (Google Authenticator, Authy)
- Strongly recommended

<Warning>Enable 2FA for maximum security. It prevents unauthorized access even if credentials are compromised.</Warning>

## Access Control

**Internal Policies**

- Principle of least privilege
- Regular access reviews
- Multi-person approval for manual data access
- All access is logged and audited

**No Access to Your Data**

- Engineers cannot view your emails
- Support requires your explicit permission
- All background checks completed

## Security Testing

**Regular Assessments**

- Annual third-party penetration tests
- Quarterly internal security reviews
- Continuous automated vulnerability scanning
- Security-focused code reviews

**Bug Bounty**
Report vulnerabilities: security@getuntyped.ai

## Disaster Recovery

**Backups**

- Continuous backups every 5 minutes
- 30-day retention
- Monthly restoration testing
- Geo-redundant storage

**Failover**

- Multi-region architecture
- Automatic failover
- Less than 30 second downtime
- Zero data loss guarantee

## Compliance

- **GDPR** - Full compliance
- **CCPA** - Full compliance
- **SOC 2 Type II** - In progress
- **PCI DSS** - Via Stripe for payments

## Incident Response

If a security incident occurs:

1. Immediate containment
2. Root cause analysis
3. Vulnerability remediation
4. User notification within 72 hours

## Best Practices

**Protect Your Account**

- Enable 2FA
- Use OAuth (Google/Microsoft) instead of passwords
- Review active sessions regularly
- Never share credentials

<Warning>If you notice suspicious activity, revoke all sessions immediately and contact support@getuntyped.ai.</Warning>

## Report Security Issues

Found a vulnerability? We appreciate responsible disclosure.

**Include**:

- Detailed description
- Steps to reproduce
- Potential impact

We respond within 24 hours and provide regular updates.
